==========================================
ENVIRONMENT VARIABLES SETUP
==========================================

Create a .env file in the root directory with these variables:

# Required: OpenAI API Key (for DALL-E image generation)
OPENAI_API_KEY=your_openai_api_key_here

# Alternative: Stability AI API Key
STABILITY_API_KEY=your_stability_api_key_here

# Required: Blockchain Configuration
NEXT_PUBLIC_CONTRACT_ADDRESS=0x0000000000000000000000000000000000000000
NEXT_PUBLIC_RPC_URL=https://sepolia.infura.io/v3/YOUR_INFURA_KEY
PRIVATE_KEY=your_private_key_for_deployment

# Network RPC URLs (for Hardhat deployment)
SEPOLIA_RPC_URL=https://sepolia.infura.io/v3/YOUR_INFURA_KEY
POLYGON_RPC_URL=https://rpc.ankr.com/polygon

# Required: WalletConnect Configuration
# Get your project ID from: https://cloud.walletconnect.com
NEXT_PUBLIC_WALLET_CONNECT_PROJECT_ID=your_walletconnect_project_id

# Optional: IPFS Configuration (uses public gateway if not set)
IPFS_API_URL=https://ipfs.infura.io:5001/api/v0
IPFS_AUTH=your_infura_project_id:your_infura_secret_key
IPFS_GATEWAY=https://ipfs.io/ipfs/

==========================================
HOW TO GET API KEYS
==========================================

1. OpenAI API Key:
   - Go to https://platform.openai.com/api-keys
   - Create a new API key
   - Copy and paste into OPENAI_API_KEY

2. WalletConnect Project ID:
   - Go to https://cloud.walletconnect.com
   - Create a new project
   - Copy the Project ID
   - Paste into NEXT_PUBLIC_WALLET_CONNECT_PROJECT_ID

3. Infura RPC URL:
   - Go to https://infura.io
   - Create a new project
   - Copy the RPC URL for your network (Sepolia, Polygon, etc.)
   - Paste into NEXT_PUBLIC_RPC_URL

4. IPFS (Optional):
   - Use public gateway (default): No setup needed
   - Or use Infura IPFS: Create project at https://infura.io
   - Or use Pinata: Sign up at https://pinata.cloud

5. Contract Address:
   - Deploy contract first: npm run deploy-contract
   - Copy the deployed address
   - Paste into NEXT_PUBLIC_CONTRACT_ADDRESS

==========================================
IMPORTANT NOTES
==========================================

- Never commit .env file to git (already in .gitignore)
- Keep your PRIVATE_KEY secure and never share it
- Use testnet for development (Sepolia, Mumbai)
